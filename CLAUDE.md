# Описание проекта

Этот проект посвящён переработке сайта стоматологической клиники с целью улучшения:

- структуры страниц
- качества текстов
- полезности для пользователей
- семантического покрытия темы
- понимания страниц современными поисковыми системами

Страницы должны быть оптимизированы для современных алгоритмов поисковых систем,
которые используют трансформерные модели (BERT, YATI и др.).

Поэтому важно:

- раскрывать сущности (entities)
- обеспечивать тематическую полноту (topical completeness)
- улучшать семантическое покрытие темы (semantic coverage)
- делать страницы максимально полезными для пользователя.

# Структура проекта

Основные файлы проекта:

### index.html
Главная страница сайта (уже переработанная).

Основная структура главной страницы:

- Hero-блок
- Услуги клиники
- Почему выбирают нас
- Популярные услуги (таблица)
- Текст о компании
- Результаты наших работ
- Технологии
- Наши гарантии
- Лицензии и сертификаты
- Врачи
- Отзывы
- Полезные статьи

Hero-блок используется **только на главной странице**.


---

### usluga-implant.html

Шаблон страницы услуги.

Страница услуги — это **внутренняя страница**, поэтому:

- hero-блок на ней использовать нельзя
- страница должна начинаться с вводного блока


---

### SEO_ANALYSIS.md

Документ с анализом SEO и объяснением доработок сайта.

Этот файл предназначен для клиента.

В нём нужно объяснять:

- какие изменения были внесены
- какие проблемы были на сайте
- как изменения улучшают сайт
- какую цель преследуют эти изменения

Важно:

Нельзя указывать конкретные цифры роста SEO или трафика.

Вместо этого нужно писать:

- каких результатов мы хотим добиться
- как изменения улучшают видимость сайта
- как они повышают полезность страниц.


---

### presentation.html

HTML-презентация для клиента.

В ней объясняется:

- какие проблемы были у старого сайта
- какая новая структура предлагается
- какие блоки были добавлены
- зачем они нужны (чтобы клиент понял)
- как они улучшают SEO и удобство сайта
- какие доработки в целом были сделаны

При добавлении новых блоков необходимо всегда объяснять:

- зачем добавлен блок
- какую проблему он решает
- как он помогает пользователям
- как он помогает SEO.

# Структура страниц услуг

Страница услуги должна иметь примерно такую структуру:

1. Вводный блок (что это за услуга)
2. Таблица с ценами
3. Показания / противопоказания
4. Преимущества
5. Этапы лечения
6. Комментарий врача
7. Сравнительная таблица (методы / технологии)
8. Методы или способы лечения
9. Рекомендации пациенту
10. Мифы об услуге
11. Вопрос-ответ
12. До / после

При необходимости можно добавлять дополнительные блоки.

# Сущности (Entities)

При создании текстов обязательно нужно раскрывать важные сущности.

Ключевые сущности зависят от тематики страницы. Например если это страница услуги про импланты, ты должен найти сущности, которые будут покрывать эту тему. 
Если это страница на тему брекетов, то сущности должны соответствовать этой теме. 

Тексты должны естественно упоминать эти сущности,
чтобы улучшить понимание страниц поисковыми системами.


---

# Принципы написания текстов

Тексты должны быть:

- понятными
- структурированными
- полезными для пациентов
- медицински корректными
- без "воды"
- без шаблонных маркетинговых фраз
- удовлетворять принцип поисковых трансформеров


---

# SEO-принципы

При создании или редактировании страниц необходимо соблюдать:

- логичную структуру блоков
- правильную иерархию заголовков (H1-H3)
- наличие таблиц и структурированных блоков
- раскрытие темы через связанные сущности
- тематическую полноту страницы

Нужно стремиться к **семантической ясности**, а не к механическому использованию ключевых слов.


---

# Правила HTML

HTML-страницы должны быть:

- чистыми
- понятными
- семантическими
- легко читаемыми
- пригодными для демонстрации клиенту

Не нужно усложнять структуру.


---

# Процесс работы

Когда Claude выполняет задачу, он должен:

1. Проанализировать текущий файл или дизайн
2. Предложить улучшенную структуру страницы
3. Написать тексты блоков
4. Создать HTML-версию страницы
5. При необходимости обновить SEO_ANALYSIS.md
6. При необходимости обновить presentation.html

Все важные изменения должны сопровождаться объяснениями.
